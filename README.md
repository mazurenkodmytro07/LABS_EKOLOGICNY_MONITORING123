Lab 3: Air Quality Monitoring System

Опис проекту

Цей проект є системою моніторингу якості повітря, що використовує індекс якості повітря (Air Quality Index, AQI) для оцінки рівня забруднення повітря в різних містах. Використовуються п’ять основних параметрів: PM2.5, PM10, NO2, SO2, O3. Система дозволяє здійснювати розрахунок AQI на основі цих показників та виводить їх у вигляді зрозумілої шкали, яка класифікує стан повітря як “добре”, “помірне забруднення”, “шкідливо” або “дуже шкідливо”.

Проект включає як бекенд, так і фронтенд частини, де бекенд відповідає за обчислення і збереження результатів, а фронтенд надає інтерфейс для введення даних і перегляду результатів.

Технології
	•	Backend: Node.js, Express, Mongoose, MongoDB
	•	Frontend: React.js
	•	Бібліотеки:
	•	Chart.js для графіків
	•	Axios для запитів API
	•	Zod для валідації даних

Інсталяція

Крок 1: Клонування репозиторію

git clone https://github.com/mazurenkodmytro07/LABS_EKOLOGICNY_MONITORING123.git
cd LABS_EKOLOGICNY_MONITORING123

Крок 2: Встановлення залежностей

Для кожної лабораторної частини, виконати наступні команди:

# Встановити залежності для lab1
cd lab1
npm install

# Встановити залежності для lab2
cd ../lab2
npm install

# Встановити залежності для lab3
cd ../lab3
npm install

Крок 3: Налаштування середовища

У корені репозиторію, створіть файл .env і додайте наступні змінні:

MONGODB_URI=your_mongodb_connection_string
PORT=3000  # для lab1
PORT=4000  # для lab3

Крок 4: Запуск проекту

# Для запуску сервера lab1
cd lab1
npm run dev

# Для запуску сервера lab2
cd lab2
npm run dev

# Для запуску сервера lab3
cd lab3
npm run dev

Опис функціоналу
	1.	Індикатор якості повітря:
	•	Користувач може ввести значення для PM2.5, PM10, NO2, SO2, O3.
	•	Система обчислює і відображає індекс якості повітря.
	•	Результати включають класифікацію якості повітря: добре, помірно, шкідливо, дуже шкідливо.
	2.	Історія розрахунків:
	•	Користувач може переглядати історію розрахунків для кожної станції.
	•	Дані зберігаються в базі даних MongoDB.
	3.	Перемикання між мовами:
	•	Можна перемикати мову інтерфейсу між українською та англійською.
	4.	Редагування нормативів:
	•	Користувач може редагувати нормативи для кожного параметра через інтерфейс.
	5.	Тести:
	•	Юніт-тести забезпечують правильність розрахунків і обробки даних.
